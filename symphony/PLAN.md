# PLAN Stage Guidance

- Produce a decision-complete implementation plan only.
- Do not edit repository files, generate artifacts, or create commits in this stage.
- State the goal, success criteria, affected packages, failure modes, validation steps, and any required follow-on updates to formal artifacts, tests, docs, and lint-sensitive code.
- If controller or service-manager behavior will change, explicitly call out the `formal/` inputs, PlantUML diagrams, property-oriented tests, and unit coverage work that implementation must include.
- Keep all git work local-only. Do not interact with GitHub or any remote git host.
