# API Generator Parity

This note records the current-service parity behavior for
`oci-service-operator-0cd.4`.

Verified parity scope:

- `database`, `mysql`, and `streaming` generator-owned API packages
- `config/samples/*.yaml` and the parity-owned entries in
  `config/samples/kustomization.yaml`
- `packages/<group>/metadata.env`
- `packages/<group>/install/kustomization.yaml`

Intentional diffs in generated Go files:

- Generated API Go files include the banner:
  `// Code generated by osok-api-generator. DO NOT EDIT.`
- Some blank-line placement and `// +kubebuilder` spacing differ from the
  current hand-maintained files, but those comment-format differences do not
  change the generated deepcopy output or CRD manifests.
- The `database` deepcopy parity check differs only in the `runtime` import
  style (`runtime "k8s.io/apimachinery/pkg/runtime"` versus the unaliased
  import in the checked-in file). The generated methods are otherwise the same.

Manual files that remain out of scope and are left untouched:

- `controllers/**`
- `api/*/*_webhook.go`
- `api/*/webhook_suite_test.go`
- `controllers/suite_test.go`
- `main.go`

Rollout-wave generators may append additional sample resources to
`config/samples/kustomization.yaml`. Parity verification therefore checks that
the `database`, `mysql`, and `streaming` sample entries remain present and in
order, rather than requiring the full file to stay byte-for-byte identical.
